# StudyHub Project Memory

## Purpose
StudyHub is a learning-focused project. The project should help organize study materials, learning tasks, and future learning workflows in a clear, practical way.

## Current Status
- Local project path: F:\projects\StudyHub
- Git branch: main
- Current structure includes: frontend, backend, docs, assets, .github
- The repo currently has local Git commits but no GitHub remote configured yet.

## Collaboration Workflow
- Git is the source of truth.
- One AI edits code at a time.
- The other AI should review, explain, or plan unless explicitly assigned as the main editor.
- Important decisions should be recorded in DECISIONS.md.
- Active work should be tracked in TODO.md.

## AI Roles
- Codex: code review, structured implementation, refactoring, Git-oriented workflow, documentation, handoff notes.
- Antigravity: IDE-assisted exploration, fast UI/project scaffolding, Google ecosystem workflows, visual iteration.

## Handoff Format
When switching from one AI to another, use:

```text
Handoff summary:
- Goal:
- Files changed:
- What works:
- What is unfinished:
- Known issues:
- Recommended next step:
```

## Project Rules
- Keep changes small and easy to review.
- Commit after each working milestone.
- Do not let two tools edit the same files at the same time.
- Prefer clear file names and simple project structure until the app direction is more concrete.
